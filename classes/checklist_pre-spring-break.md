## What's In Each Submission

### 1. Team Formation & GitHub Setup (Feb 6)
- [ ] Team roster with UNIs
- [ ] Team name
- [ ] GitHub repo fork set up

### 2. User Interview Materials (Feb 17)
- [ ] Interview script / prep docs
- [ ] Raw interview notes (5+ interviews per team)
- [ ] Post-interview snapshots (quick summaries)

### 3. Product Brief, Brand Position & Synthetic Testing (Feb 19)
- [ ] Product Brief (use template from `assignments/20260219_synthesis-brief-position/`)
- [ ] Brand Position document (use template from same folder)
- [ ] Synthetic user testing results (5+ rounds with copy iterations)

### 4. Landing Page + Final Synthetic Testing (Feb 24)
- [ ] Updated Brand Position (refined from Feb 19 draft)
- [ ] Landing page copy (hero, how it works, social proof, objections, CTA)
- [ ] Synthetic testing results (2+ rounds: V1, V2, V3 if possible)
- [ ] **Live deployed URL** (Carrd, Framer, Webflow, Vercel, Netlify, or raw HTML)

### 5. Google Analytics Certification (Feb 26)
- [ ] Proof of completion (screenshot or certificate)
- [ ] Complete via [Google Skillshop](https://skillshop.google.com)
- This is an individual assignment, not a team submission

### 6. MVP Scope & Dev Plan (Mar 5)
- [ ] MVP scope document:
  - Core flow (user action to value)
  - What features are cut
  - What's faked (Wizard of Oz / concierge)
  - Team roles and ownership
  - Tech stack
- [ ] CLAUDE.md file (team's shared AI instructions, checked into git)
- [ ] Git workflow set up and working

### 7. MVP Presentation (Mar 12)
- [ ] 5-minute live demo in class
- [ ] Peer feedback forms (evaluating other teams)
- See the Demo Day guide for presentation format
